7. Token Economy Protocols
This is the operational spine. Every rule below exists to minimize tokens-per-decision.
7.1 Hub Density Rules
**Tables over prose.** Anything enumerable → table.
**One fact per line.** No multi-clause sentences.
**No filler.** Strip "this section describes," "the purpose of," "in order to."
**Backticks for symbols.** Always. Saves disambiguation tokens.
**Acronyms after first definition.** Define once in `GLOSSARY`, then reuse.
**Numbers, not words.** "20" not "twenty."
7.2 Spoke Density Rules
**No introductions.** Header → invariants. No "this spoke covers..."
**Code over commentary.** A 5-line example beats a 5-sentence explanation.
**Cross-ref, don't duplicate.** If a fact lives in another Spoke, link, don't restate.
**Kill examples that earn no rule.** If an example doesn't lock down a constraint, delete it.
7.3 Loading Discipline
**Lazy by default.** Spokes load only on activation match.
**One-and-done.** A Spoke loaded once per session is not re-read.
**Hub > Spoke for boot.** If the Hub answers it, never load a Spoke.
**Reject "just in case" loads.** If unsure a Spoke is needed, ask the user before loading.
7.4 Save-State Discipline
**Diff, not rewrite.** Save-state always proposes a unified diff. Never re-emits the full file.
**Smallest viable edit.** One section changed = one section in the diff.
**No commentary in the file.** Explanations go in chat, not in
project_brief.md (/c:/nbs/abible/project_brief.md)
.
**Archive purges, not duplicates.** When evicting from Hub → archive, the entry leaves the Hub entirely.
7.5 Conversation Discipline (agent-side)
**No re-stating context.** I do not echo the Hub back to you mid-session.
**No progress narration.** "Now I will..." is banned. Just do it.
**Confirm completion in ≤2 sentences.** Diff already speaks for itself.
**Block early.** If I detect a conflict, I stop *before* doing the work, not after.
**Question batching.** If I need clarification, batch all questions into one turn.
7.6 Anti-Bloat Tripwires
Hub > 400 lines → halt edits, propose archive sweep.
Spoke > 150 lines → halt edits, propose split.
`DECISIONS` > 20 → archive oldest before adding new.
`SESSION LOG` > 10 → archive oldest before adding new.
Same fact appears in Hub + Spoke → halt, deduplicate to Spoke, leave Hub pointer only.
7.7 Cost-Aware Defaults
When a task can be done with a Spoke load *or* an interactive question, **prefer the question** if the Spoke is > 80 lines and the question is < 3 lines.
When summarizing chat, prefer pruning to compression — drop irrelevant turns rather than re-encode them.
Never paste a full file into chat as "context" if a path + line range will do.
