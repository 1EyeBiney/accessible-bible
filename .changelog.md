
## 0.9.0 - Context-Aware Verbosity and Location Hotkey
### Added
* Location Hotkey: Mapped the `Tab` key to a "Where am I?" function that forces a full structural announcement (Book, Chapter, Verse) without moving the index.
* Smart Verbosity: The engine now tracks the `lastAnnouncedBook` and `lastAnnouncedChapter`. Sequential reading only announces the verse number unless a chapter or book boundary is crossed.
### Changed
* `gemini.md`: Updated Section 3 (Keybindings) to document the `Tab` key and Section 1 (Prime Directive) to mandate contextual audio verbosity.
0.7.4 - Splash Screen Latch Fix
### Added
* Splash Screen Focus Latch: Added `role="application"` to the `#splash-screen` container to force NVDA/JAWS into Focus Mode immediately on page load, capturing the `Enter` key automatically.
### Changed
* gemini.md: Documented the requirement for the splash screen to have an application role to prevent Browse Mode trapping.
## 0.7.3 - Splash Screen Quarantine and Pre-Init Focus Trap
### Added
* DOM Quarantine: The main application container is now set to `display: none` on load, ensuring the screen reader has absolutely no other text to read except the initialization button.
* Pre-Init Focus Trap: Added a `blur` listener directly to the initialization button. If focus is lost before the engine starts, it immediately snaps back to the button.
### Changed
* Restored `autofocus` to the initialization button since the DOM quarantine eliminates the "Screen Reader Brawl" on page load.
* Updated `activateEngine()` to reveal the main application container right before shifting focus to the main engine trap.
## 0.7.1 - Splash Screen Audio and Focus Fix
### Added
* Auto-Focus: Added the `autofocus` attribute to the initialization button to ensure it captures the 'Enter' key immediately on page load.
* Wake-up Call: Implemented a 1-second delayed announcement in `app.js` to force screen readers to vocalize the "Press Enter" instruction after the page settles.
* ARIA Labeling: Added `aria-label` to the initialization button to ensure the prompt is read even if the live region fails to trigger.
### Fixed
* Resolved "Silent Load" issue where the app appeared unresponsive because no element had initial focus.
