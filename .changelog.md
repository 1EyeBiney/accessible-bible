##0.11.0 - Zero-Latency Search Mission
### Added
* Stealth Search Input: Injected a visually hidden `<input type="text">` (`#search-input`) to capture search queries natively without breaking the Focus Trap architecture.
* Instant Memory Filtering: Built a high-speed search function that filters the `memoryCache` and populates a state-based `searchResults` array.
* Search Carousel: Mapped `[` (Previous) and `]` (Next) keys to cycle rapidly through active search results. 
* Contextual Drop-in: Viewing a search result automatically updates the main index, allowing immediate context reading via the Left/Right arrows.
* Search Status: Updated the `S` key to append active search statistics (e.g., "Search active: viewing match 4 of 42") if a search is loaded.
### Changed
* Mode Safety: `clearAllModes()` now safely dumps the `searchResults` array when navigating away via coordinate jumps (`B`, `C`, `V`, or Page keys).
* `gemini.md`: Updated Section 3 (Keybindings) to include `F`, `[`, and `]`. Updated Section 4 to mandate the Stealth Input Protocol for searches.
## 0.10.0 - Ambient Audio Engine and Grab Bag Playlist
### Added
* Ambient Audio Engine: Implemented a dual-buffer audio system to support seamless crossfading between background tracks.
* Grab Bag Playlist: Added a shuffle-and-pop algorithm for the 10 hymn assets to ensure non-repeating playback.
* Auto-Play Initialization: The ambient music now begins playing automatically at volume level 10 once the application is initialized.
* Volume Control: Mapped `Shift + V` to cycle through ambient volume stages (0, 5, 10, 20, 30, 40).
* Next Track: Mapped the `N` key to manually trigger a crossfade to the next random track.
### Changed
* Key Re-mapping: Moved the "Digital Margins" note-taking mission from `N` to `M` (Memo) to accommodate the new audio controls.
* `gemini.md`: Updated Section 3 (Keybindings) to reflect the new `M`, `N`, and `Shift + V` assignments.
